### 観点B：AI文型・構造の癖

以下の文型が出ていないか確認：

1. **冒頭の問いかけ三連**：「〜なお悩みはありませんか？」「〜でお困りではありませんか？」
2. **3点並列の体言止め**：「丁寧なカウンセリング・確かな技術・安心のアフターケア」型（中点で3つ並べる）
3. **抽象→具体テンプレ**：「お一人おひとりに最適な〜をご提案。例えば〜」
4. **結びの誘導定型**：「あなたも〜してみませんか？」「お気軽にお問い合わせください」
5. **中点「・」の過剰使用**：1ページ2回以上出てきたらAI臭
6. **「お客様の声」AI生成丸出し**：「丁寧な対応で安心できました！（30代女性）」型 — 疑念から始まっていない・実名イニシャル無し・職業無しは×
7. **冗長な「させていただきます」「いたします」連発**
8. **「〜とは、〜のことです。」型の定義文連発**

**人間味チェック**（1ページに最低5つ仕込まれているべき）：
- [ ] 一人称（私／うちの院／鈴木です）が文中に1回以上
- [ ] 固有名詞・固有数字（のべ◯名／半蔵門徒歩4分／腸腰筋 等）
- [ ] ためらい・限定（「全員には効きません」「合う合わないあります」）
- [ ] 断言（形容詞を削って「治します」じゃなく「ボキボキしません」と言い切る）
- [ ] 口語の挟み込み（「で、」「ぶっちゃけ」「ですよね」）
- [ ] 失敗談・反対意見（「最初の3年は集客できなくて」）
- [ ] 読点で粘る長文が1セクションに1つ

報告フォーマット：
```
[セクション名] [問題のあるパターン] - 該当箇所："..." → 修正方向："..."
```

### 観点C：文脈破綻・一貫性

以下の整合性を確認：

1. **人物名の一貫性**：架空人物（例：salon の施術者名）が複数箇所で一致しているか、性別・年齢・資格・経歴の矛盾がないか
2. **数字の一貫性**：実績数（のべ◯名）、年数（10年）、価格（初回◯円）、所要時間（60分）がページ内で揺れていないか
3. **地名・住所の一貫性**：「西淀川区」と「千舟3丁目」のように、地理的に整合する地名同士か（架空でも県内で完結しているか）
4. **時系列の整合**：「3ヶ月で」と「半年後」のように、ターゲットに対する時間感覚が破綻していないか
5. **CTA文言の整合**：FVと最終CTAで「LINE予約」と「予約フォーム」など主CTAが揺れていないか
6. **価格の整合**：FV／料金セクション／最終CTAで初回価格が一致しているか
7. **業態の一貫性**：「整体院」「整骨院」「鍼灸院」を混在させていないか（凪は整体院想定）
8. **ターゲット女性向け文脈で、男性向け語彙が混入していないか**（または逆）

報告フォーマット：
```
[箇所A行 vs 箇所B行] [矛盾の種類]："Aでは◯◯、Bでは△△" → 修正案："..."
```

### 観点D：法的禁則（業種別・最重要）

#### gym
- 「絶対痩せる」「100%成功」「全員結果が出る」 → 景表法・優良誤認
- 「医師推奨」（根拠なし）
- ビフォーアフターの「個人差注記」漏れ
- 「業界No.1」（客観的根拠なき最大級）

#### salon ★最重要
```
# 絶対NG
治る, 治療, 治癒, 完治, 根治, 根本改善, 根本治療,
診察, 診療, 患者, 効果, 効きます, 必ず, 絶対, 100%,
医師推奨, 医学的根拠, 日本一, No.1, 業界一,
ヘルニア改善, 椎間板, 自律神経失調症（疾患名と改善をセットで使うのは禁止）
```
- ※「肩こり」「腰痛」「症状」自体はOK、ただし「治る」「改善する」と並べない
- **ビフォーアフター写真は載せない**（医療広告ガイドライン準じNG運用）
- お客様の声・通院モデルに「※効果には個人差があります」明記
- **二重価格表示**（通常¥8,800→初回¥4,980）は「過去価格8週間以上の販売実績」根拠を注記

#### coach
```
# 絶対NG
絶対, 100%, 必ず, 業界No.1, 日本一, 最高, 最大級, 史上初,
全員成功, 誰でも必ず, 最短最速,
治る, 改善する（心身症状）, 治療, 効果保証,
保証します（条件なしで）, 完全成果報酬
```
- 「年収◯倍」断定は受講生の声でも避ける（または個人差注記）
- **特商法表記**へのリンクが必須（デモなら `href="#"` でOKだがリンクは置く）
- 「※成果には個人差があります」を受講生インタビュー各カードに添える

報告フォーマット：
```
[行番号] [禁止語] - 文脈："..." → 修正必須："..."
[セクション] [注記漏れ] - "..." を追加すべき
```

### 観点E：肖像権・デモ表記・構造化データ

1. **Unsplash等のフリー素材の人物写真が、架空人物のプロフィール枠に使われていないか**
   - 該当箇所：施術者顔写真／お客様の声の顔写真／受講生インタビュー写真
   - 検出：`grep -nE "unsplash|pexels|pixabay" demos/<target>.html`
   - 正解：`#1a1a1d` or `#f5f1ea` 単色背景＋ストライプテクスチャ＋「撮影予定」スパン

2. **デモ表記の3層が全て揃っているか**
   - [ ] Layer 1：page top の sticky banner（`<div class="demo-banner">`）
   - [ ] Layer 2：title / meta / OGP に【デモLP】プレフィックス
   - [ ] Layer 3：フッターに demo-disclaimer（「写真の人物は本文中の人物と一切関係がありません」明記）

3. **構造化データに `LocalBusiness` / `Organization` / `Person` / `Service` が出ていないか**
   - 出ていたら**架空事業を Google に実在として登録するリスク**
   - 正解：`WebPage` schema のみ、`isPartOf` に aiweb-lab を指定

4. **`noindex` 指定の有無**（`<meta name="robots" content="noindex,...">`）

5. **著作権年が 2026 か**（`© 2025` は古い）

6. **`canonical` URL の整合**（noindex なら canonical 不要 or 自身を指す）

報告フォーマット：
```
[項目] [状態：OK/欠落/誤り] - 詳細："..."
```

## 4. 監査の進め方

1. まず対象ファイルを `Read` で全体把握（ファイルサイズが大きい場合は分割読み）
2. `Grep` で禁止語を一気に検出（観点A・D）
3. セクション順に観点B・Cをチェック
4. 観点Eは Head 部分と Footer 部分を集中読み
5. 報告作成、修正は実施しない

---

## 5. 注意事項

- **修正の実装はしない**。レポートだけ。
- 同じ問題を観点をまたいで重複報告しない（例：AI臭ワード「寄り添い」と人間味チェック「ためらいなし」は別物として扱う）
- 「これは指針違反だが意図的にやっている可能性がある」場合は致命度を下げて報告
- **デモであることを踏まえる**：実在クライアント納品物として致命的でも、デモ表記済みなら致命度を1段下げる（例：「営業時間 9-22時」がダミー風でも、デモ明示済みならC級）
- 章番号や雑誌風演出（「Vol. 04 — Spring 2026」等）は世界観なのでAI臭ではない

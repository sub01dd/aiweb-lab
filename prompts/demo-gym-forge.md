## 4. AI感を消す原則（最優先）

### 禁止語（必ず置換すること）

| AI語 | 置換 |
|---|---|
| 最適な／最高の／究極の | （削除して固有数字や具体名へ） |
| 圧倒的な／本格的な／徹底的に | のべ◯人／競技経験者がやってる／1ミリ単位で |
| 確かな技術／確かな実績 | のべ◯人診た／12年やってる |
| 質の高い／豊富な経験／きめ細やかな | （具体ディテール、3つ列挙、固有工程名） |
| お客様一人ひとりに寄り添い | あなた／田中さん（仮名）／横で見てます |
| ライフスタイル／ニーズ／ソリューション／メソッド | 暮らし／困りごと／やり方／◯◯式 |
| 〜を実現します／〜を提供します／〜をサポートします | 〜できます／〜します／一緒にやります |
| お客様の声を大切にしています／〜に寄り添います | （削除） |
| ワンランク上の／至福の／上質な | （削除） |
| 結果にコミット風 | （ライザップ模倣回避、独自言語で） |

### 禁止文型

1. **冒頭の問いかけ三連**「〜なお悩みはありませんか？」型 → 「3ヶ月で5kg落としたい。でも続かない。わかります、私もそうでした。」型へ
2. **3点並列の体言止め**「丁寧なカウンセリング・確かな技術・安心のアフターケア」 → 削るか、最低1つは口語で崩す
3. **抽象→具体テンプレ**「お一人おひとりに最適なプランをご提案。例えば〜」 → いきなり具体ケースから始める
4. **結びの誘導定型**「あなたも理想の身体を手に入れませんか？」 → 削除して「電話、9時から20時まで出ます。鈴木が出ます。」型へ
5. **中点「・」濫用** 1ページに1回まで
6. **「お客様の声」AI生成丸出し**：「丁寧な対応で安心して通えました！(30代女性)」 → 「正直、最初は怪しいと思ってました。3回目で諦めて、4回目で『あ、効いてる』ってなった。(田中さん・34・経理)」型

### 人間味を出す7つの手筋（1ページ内に最低5つ仕込む）

1. **一人称**：「私」「うちのジム」「鈴木です」を文中に最低1回
2. **固有名詞・固有数字**：「のべ1,247人」「半蔵門駅から徒歩4分」「ジャイアントセット法」
3. **ためらい・限定**：「全員には効きません」「正直、合う合わないあります」
4. **断言**：形容詞を削ってでも「落とせます」「鍛え直します」と言い切る
5. **口語の挟み込み**：「で、」「ぶっちゃけ」「ですよね」を要所に1つ
6. **失敗談・反対意見**：「最初の半年は全然集客できなくて」
7. **読点で粘る**：句点で短く切らず、読点で息を作る文を1セクションに1つ

### ビジュアルAI臭の禁止

- 中央寄せ3カラム×等間隔の「特徴3つ」テンプレ → ジグザグ配置や非対称分割（5:7）へ
- shadow-md（0 4px 6px）の量産 → 影は1箇所だけ濃く、他は罫線で
- border-radius 16px の全要素統一 → カード6/ボタン4/写真0
- fade-in-up全乗せ → ヒーローの1箇所だけ。スクロール時は静止
- hover時 scale(1.05) 量産 → 色変化200msか下線伸長のみ
- unDraw/Storyset風イラスト → 写真＋手描き丸囲み（SVGで線をブレさせる）
- AI生成人物（指の融合、耳の左右非対称、歯の枚数異常） → **人物は実写優先、AI生成不可**
- 紫×シアン/ティール×オレンジのグラデ → 禁止
- backdrop-blur-xlのガラス風 → 禁止
- 全枠線が同じ輝度・同じ太さ → 太さ・色に差をつける
- 「導入1,000社」風の根拠なきカウントアップ → 禁止

## 5. 和モダン指針（鋼バリアント）

### レイアウト

- グリッド12カラム、コンテンツ最大幅 1120〜1200px
- セクション縦余白：PC 120〜180px / SP 80〜120px、**意図的に不揃いに**（py-16/py-24/py-32を使い分ける）
- 非対称分割（5:7など）を1セクションは入れる
- 写真は端まで断ち切る（bleed）を1箇所
- 余白の中に小さなメモ書き（「※カウンセリング当日は私服でOK」を斜め-3degで添えるなど）

### 装飾

- 罫線は0.5pxか1pxどちらかに統一、色は `--border` で2段階
- 紋様（青海波）はフッター帯1箇所、高さ24〜40px、不透明度8〜12%
- 紋様タイル背景・赤い印鑑風スタンプ・桜扇梅マーク・暖簾写真は禁止
- SVGノイズ3〜5% を背景に重ねて紙質感（鋼バリアントでは控えめ）
- バッジは-3degの回転を1箇所だけ（実績バッジ等）

### タイポ細部

- 句読点ベタ詰めで重い箇所は `font-feature-settings: "palt"` をかける
- ルビは font-size 50%
- 数字（体重・体脂肪・回数）は和文より大きく、Inter 700で

### アニメーション

- fade-in + translateY(20px) を 700ms ease-out のみ
- ヒーローのみ控えめパララックス（10%以内）
- カーソル追従・パーティクル・ローディングアニメ・全要素reveal は禁止

### 写真ディレクション

- **人物実写優先**（Trainerは特に。AI生成不可）
- 抽象的なヒーロー背景はAI可（Midjourney等）、ただし `clean / modern / minimalist` 禁句、`35mm film / grain / shadow / low saturation` 指定
- Before/Afterは同一ポーズ・同一照明・同一距離、体重/体脂肪率/期間明記、本人コメント付き
- ストックの「握手」「PCを指差すスーツ」「外国人モデル白背景」禁止
- カラーは彩度-20〜-40、シャドウ持ち上げない（陰を残す）
- **トレーナー実写写真がない場合は、コメント `<!-- TODO: 撮影写真差し替え -->` を入れた灰色プレースホルダーで -- ただし顔出しダミーは入れない**

## 6. セクション構成

### 固定ヘッダー
- ロゴ（左）／ナビ（FORGE/メソッド/料金/トレーナー/お客様の声/アクセス）／右にCTA `[無料カウンセリング]`

- モバイルはハンバーガー → 全面オーバーレイ、最下部にCTA

### 01 FV（ヒーロー）
- 背景：黒。トレーナーが鏡前に立つ斜光モノクロ写真を画面端まで bleed
- H1：「鍛えるな、鍛え直せ。」（Shippori Mincho 800 / 72px / 改行制御）
- サブ：「経営者のための、2ヶ月集中ボディメイク。」
- 実績バッジ3つ（横並び）：のべ◯人指導 / 目標未達は全額返金 / 完全個室・スーツ預かり
- CTA2つ：`[無料カウンセリングを予約]`（煉瓦）／ `[LINEで料金を見る]`（透過枠線）
- 下にスクロール矢印「↓ SCROLL」（細い罫線）
- パララックスは10%以内

### 02 悩みの言語化
- セクション見出し：「会食、出張、寝不足。それでも、戻したい。」
- 共感3行（チェックリストではなく一人称の独白）：
  - 「ジャケットの肩がつかえる。」
  - 「鏡の自分を、もう撮りたくない。」
  - 「土日にジムへ行く時間が、もうない。」
- ※「こんなお悩みはありませんか？」型は禁止

### 03 FORGEの約束（コミット）
- 「FORGEが、お返しするもの。」
- 約束3つ（横並び3カラムは禁止 → 縦のジグザグor非対称グリッド）：
  - **数字で出します**：InBody週次、栄養数値、体組成の推移をデータで返す
  - **未達なら、返します**：契約時の目標体組成に届かなければ全額返金（条件は契約書に明記）
  - **担当は、変えません**：初回カウンセリングから卒業まで同じトレーナーが見る
- 各約束の説明文は120〜180字、語尾「。」で締める

### 04 メソッド（データドリブン）
- 「鍛え直す、3つの工程。」
- 工程① InBody計測：週1回、体組成を数値で見る
- 工程② 栄養管理：LINEで食事写真→マクロ栄養素を返す、運用画面キャプチャを入れる
- 工程③ セッション：60分×週2、フォーム動画記録→次回比較
- 各工程に手書きSVG矢印で繋ぐ（量産アイコンライブラリは避ける）

### 05 Before / After
- 6〜8名、同条件撮影。各カードに：
  - 期間（例：8週間）
  - 体重 -◯kg / 体脂肪率 -◯% / 筋量 +◯kg
  - 本人の一言（120字、口語で）
- ※モデル流用・加工強めは禁止。実写準備できない場合はプレースホルダー＋撮影TODO
- 「個人差があります」を控えめに添える（炎上回避）

### 06 トレーナー
- 大きめポートレート1枚（顔出し）
- 経歴：年齢／NSCA-CPT／競技歴／指導実績数（固有数字で）

- ※「詳しくはカウンセリングで」隠す型は禁止

### 09 店舗

### 10 お客様の声

## 8. CTA設計

### 階段
1. 冷：LINE登録で料金PDF配布（「友だち追加で30秒で届きます」）
2. 温：無料カウンセリング予約（60〜90分・勧誘なし明記）
3. 熱：体験トレーニング込みカウンセリング（90分）

### LP内配置
- FV／約束直後／メソッド直後／Before/After直後／料金直後／最終 の **6箇所**
- モバイルは下部に追従CTAバー（LINEと予約の2ボタン）
- PCは右下に縦書き「無料カウンセリング」追従

### CTA文言（禁止と推奨）
- 禁止：「お問い合わせ」「送信」「申し込む」

## 11. 技術要件

### ファイル構成
```
demos/
  gym.html               # このプロンプトの出力対象
assets/
  css/style.css          # 既存。必要なら gym.html 用に追加クラス
  js/main.js             # 既存。必要なら追加
  img/demos/gym/         # 画像（撮影前提でプレースホルダー）
```

- `gym.html` は **単独で完結**（外部CSSは既存style.cssの上書きで対応）
- 既存の自社サイト `index.html` のヘッダー・フッター・ナビは流用せず、デモ独自で組む

### HTML
- セマンティック（header/main/section/article/footer）
- 構造化データ（LocalBusiness + Service）
- OGP/Twitter Card完備
- `lang="ja"`、適切な `meta description`

### CSS
- CSS変数で色管理（上記の `:root` 定義）
- モバイルファースト、ブレークポイント 768 / 1024 / 1280
- グリッドは CSS Grid、必要に応じて Flexbox
- セクション余白は意図的に不揃いに（`--space-section-sm/md/lg` を使い分け）
- アニメは IntersectionObserver で fade+translateY(20px) のみ、ヒーローを除いて

### JavaScript
- バニラJSのみ（フレームワーク導入不可）
- 機能：
  - ハンバーガーメニュー
  - FAQアコーディオン
  - reveal（IntersectionObserver）
  - スムーススクロール
  - 追従CTAの表示制御

### パフォーマンス目標
- Lighthouse Performance / SEO / Accessibility 全て90+
- 画像は WebP、`loading="lazy"`、`decoding="async"`
- フォントは `preload` ＋ `display=swap`
- メインスレッドブロック < 200ms

### アクセシビリティ
- コントラスト WCAG AA以上（煉瓦は本文不可、CTA背景のみ）
- フォーカスリング可視
- ハンバーガー、FAQ、フォームに適切な aria-*
- prefers-reduced-motion 対応（reveal無効化）
